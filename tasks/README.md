# タスク管理

実装 TODO はこの `tasks/` ディレクトリ配下で、**設計 → 実装 → デプロイ** の
フェーズごとにサブディレクトリを分けて管理します。ファイル 1 つ = タスク 1 件です。

## フェーズ構成

| フェーズ | ディレクトリ | 内容 |
| --- | --- | --- |
| 設計 | [design/](./design) | 要件ヒアリング・設計書([docs/](../docs))の作成 |
| 実装 | [implementation/](./implementation) | 設計に基づく TDD 実装(frontend/ の作成〜E2E テスト) |
| デプロイ | [deploy/](./deploy) | Docker 化・本番構成・運用方針の確定 |

全タスクを一覧で確認するには [TASKS.md](./TASKS.md) を参照してください。
各フェーズの目的や進め方は、それぞれのディレクトリの `README.md` を参照してください。

## ファイル命名・フォーマット

`NNN-短い識別子.md`(`NNN` はフェーズ内での 3 桁連番。実行順の目安)

```markdown
# NNN: タイトル

- ステータス: 未着手 | 進行中 | 完了 | 保留
- 関連文書: [設計書へのリンク]

## 概要

(何を・なぜ)

## 完了条件

- [ ] ...
```

## ステータスの意味

| ステータス | 意味 |
| --- | --- |
| 未着手 | まだ手を付けていない |
| 進行中 | 着手済み・未完了 |
| 完了 | 完了条件を満たした |
| 保留 | 決定待ちの事項があり着手できない([要件不足事項一覧](../docs/schedule-ui-open-questions.md)参照) |

## 全体進捗

→ [TASKS.md](./TASKS.md)
