typescript-lsp が導入できなかった理由

他のプラグインは .claude-plugin/plugin.json を持つ標準的なプラグイン構成だが、 typescript-lsp は marketplace.json 内の lspServers フィールドとして直接定義される特殊な形式で、実体となるファイル一式が存在しない。 APMの汎用インストーラ(apm.yml/SKILL.md/plugin.jsonの いずれかを探す)では検出できずエラーになるため、Claude Code純正の /plugin install typescript-lsp@claude-plugins-official で個別に導入する必要がある。

vercel/next.js 由来のSkillでハマった点

apm install はリポジトリのデフォルトブランチを main/master 決め打ちで試すため、デフォルトブランチが canary の vercel/next.js では vercel/next.js/skills/next-dev-loop のような素の指定だと 「not accessible or doesn't exist」で失敗した。 vercel/next.js/skills/next-dev-loop#canary のように #<ブランチ名> を明示することで解決した。
