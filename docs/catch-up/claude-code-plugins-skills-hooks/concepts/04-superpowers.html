superpowers(TDD特化・高star)

GitHub: github.com/obra/superpowers · 252,008 stars / 22,496 forks(2026-07-11時点) · 2025年10月作成、活発に開発中(直近プッシュも当日) · サードパーティ製だが Anthropic公式マーケットプレイスに特定コミットSHAで採録されている

説明

superpowers は「composable skillsの上に構築された、コーディングエージェント向けの 完全なソフトウェア開発方法論」を謳うプラグイン。単発のSkillではなく、 要求分析→設計レビュー→実装計画→実装、という一連の開発プロセス全体をカバーする。

動作の流れ

  1. コードを書き始める前に、まず「本当は何を実現したいか」をユーザーに問い返す
  2. 会話から引き出した仕様を、読みやすい単位に分けて提示し、ユーザーの承認を得る
  3. 承認後、true red/green TDD・YAGNI・DRY を重視した実装計画を立てる
  4. 「go」の合図で subagent-driven-development(サブエージェントが各タスクを実行し、 組み込みのコードレビューで相互チェックしながら進める)を開始する

本プロジェクトとの親和性

  • TDD: テスト設計書で 「TDD、3層のテストピラミッド。最も下のレイヤーから先にテストを書く」と明記されており、 superpowersのtrue red/green TDD方針とそのまま合致する
  • YAGNI/DRY: 本リポジトリの CLAUDE.md が 「追加機能や抽象化を要求以上に持ち込まない」「3つの似たような行の方が早すぎる抽象化より良い」 と明記しているのと同じ思想
  • サブエージェント駆動開発: settings/schedules.json の ファイル同期の安全規則(バリデーション→アトミック書き込み→直列化→バックアップ)のような 妥協不可な要件を、複数タスクに分けて相互レビューしながら実装するのに向いている

インストール方法(参考)

# Anthropic公式マーケットプレイス経由(推奨)
/plugin install superpowers@claude-plugins-official

注意

  • 「開発プロセス全体」を扱う性質上、既存の CLAUDE.md の指示(TDD・YAGNI等)と 重複・競合しないか、導入前に確認したほうがよい
  • star数が9か月で25万を超えるという急成長ぶりは、2026年のAIエージェントツール界隈の 盛り上がりを踏まえても際立って大きい。Anthropic公式マーケットプレイスに コミットSHA固定で採録されている点を信頼性の裏付けとして重視した (単独のGitHub star数だけで判断していない)

確認問題

  1. superpowers の開発フローを4段階で説明してください。
  2. 本プロジェクトの CLAUDE.md や テスト設計書と、 superpowers の思想が一致している点を2つ挙げてください。
