Concept 01 / 05

用語整理と調査方針

説明

agent-tool-management/ では 「MCP/Skillを発見・インストール・依存管理するツール」自体を比較した。本教材はその一段下、 「実際にそのツールへ乗せる中身」である Plugin・Skill・Hook を、 このリポジトリ(Next.js + React + TypeScript のスケジュール編集UI、TDD前提、 devcontainer + Claude Code)に有用そうなものに絞ってカタログ化する。

3つの用語

用語説明
Skill SKILL.md という手順書。特定の状況でエージェントが自律的に読み込む (または /name で明示的に呼び出す)。例: 「Reactのパフォーマンス最適化パターン」
Plugin Skill・Agent(サブエージェント)・Hook・MCPサーバー・LSPサーバーなどをひとつの ディレクトリにまとめた配布単位。/plugin install でチームに配れる (詳細は agent-tool-management の Claude Code Plugins & Marketplace)
Hook ツール呼び出しの前後(PreToolUse/PostToolUse/Stop等)に 自動実行されるシェルコマンド。例: ファイル編集後に自動で prettier をかける

調査方針

  • 公式または高star数のみ対象: Anthropic公式マーケットプレイス (anthropics/claude-plugins-official)、Vercel/Next.js公式リポジトリ、 またはGitHub star数が数千〜数十万規模の実績あるものに限定する
  • 信頼性の確認方法: マーケットプレイス掲載物は author フィールドで 一次配布元を確認。サードパーティは GitHub API で star数・fork数・直近更新日・ アーカイブ状態を機械的に確認した(詳細は docs/catch-up/claude-code-plugins-skills-hooks/research.md)
  • APM(apm.yml)への追加は行わない: 本教材はカタログ化のみ。 実際に導入する場合は、必要になったタイミングで個別に検討する

確認問題

  1. Skill・Plugin・Hook の違いを一言ずつで説明してください。
  2. 本教材が「公式または高star」に絞った理由と、その確認方法を説明してください。
