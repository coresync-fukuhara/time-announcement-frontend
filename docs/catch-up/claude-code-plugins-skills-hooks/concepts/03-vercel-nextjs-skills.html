Skill概要本プロジェクトでの用途
react-best-practices Vercel Engineeringによる React/Next.js パフォーマンス最適化ガイドライン (バレルインポート・非同期ウォーターフォール・不要な再レンダリング等69項目をチェック) 時刻グリッドの5分刻みトグルボタン(多数のコンポーネントが並ぶUI)の再レンダリング最適化に直結

vercel/next.js の /skills(140,802 stars, 本体リポジトリに統合)

GitHub: github.com/vercel/next.js/tree/canary/skills · 以前は別リポジトリだったが、フレームワーク本体とバージョン追従させるため vercel/next.js 本体に統合された

Skill概要本プロジェクトでの用途
next-dev-loop 実行中の next dev に対し、/_next/mcp(Next.jsの視点)と ブラウザ操作(ブラウザの視点)を組み合わせて「型/ビルドが通るだけでなく 実際に動くこと」を検証する 本プロジェクトが既に導入している Next.js DevTools MCP + Playwright MCP の組み合わせと、まさに同じ発想。編集→検証のループを型式化するのに直結する
next-cache-components-adoption Cache Components(cacheComponentsフラグ)を有効化し、 発生するブロッキングルートを解消する 現時点では1画面構成の小規模UIのため優先度は低いが、将来の最適化フェーズで検討候補
next-cache-components-optimizer Cache Components導入後の初回表示・画面遷移を最適化する2つの診断ループ 同上(Cache Components導入後に初めて意味を持つ)

インストール方法(参考)

# Vercel Skills(skills.sh)経由でのインストール一般形
npx skills add vercel-labs/agent-skills --skill react-best-practices
npx skills add vercel/next.js --skill next-dev-loop

ポイント

  • next-dev-loop は本プロジェクトが MCP サーバー導入設計書で選定した Playwright MCP・Next.js DevTools MCP の使い方そのものをSkill化したものであり、 4候補の中で最も直接的に関連が深い
  • react-best-practices・composition-patterns はUI実装の設計段階から、 web-design-guidelines は実装後のレビュー段階から、それぞれ活用できる
  • Cache Components関連の2つは、現状の1画面構成では優先度が低い(将来の拡張時に再検討)
