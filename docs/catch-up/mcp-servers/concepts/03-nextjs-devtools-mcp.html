Next.js DevTools MCP

説明

Next.js 16 以降は、開発サーバー起動時に /_next/mcp という MCP エンドポイントを内蔵している。 next-devtools-mcp(vercel/next-devtools-mcp)は、 このエンドポイントを見つけて中継するだけの薄いコネクタで、 エージェントはこれを通じて実行中の Next.js アプリの内部状態(ビルドエラー・ルーティング情報など)を参照できる。

例えるなら、工場の生産ラインに最初から設置されている診断ポート(/_next/mcp)に、 診断ケーブル(next-devtools-mcp)を挿して現場の状況を読み取るイメージ。 ケーブル自体は情報を作らず、ラインが持っている情報をそのまま橋渡しする。

本プロジェクトでは BFF(Route Handlers)を含む Next.js アプリを開発するため、 「ビルドは通るが挙動がおかしい」といった調査で、エージェントが ブラウザのコンソールログやNext.js自身のエラー情報に直接アクセスできるようになる。

コード例(設定)

# 最も簡単な追加方法
npx add-mcp next-devtools-mcp@latest

# Claude Code CLI で追加する場合
claude mcp add next-devtools npx next-devtools-mcp@latest

// .mcp.json
{
  "mcpServers": {
    "next-devtools": {
      "command": "npx",
      "args": ["-y", "next-devtools-mcp@latest"]
    }
  }
}

ポイント

  • Next.js 16 以上が必須(/_next/mcp エンドポイントがこのバージョン以降にのみ存在)。 本プロジェクトは新規構築のため、最新版採用であれば要件を満たす
  • 複数ポートで動く複数の Next.js インスタンスも自動で見つけて接続できる
  • あくまで開発サーバー(next dev)向けの機能。本番ビルド/本番運用には影響しない
  • Playwright MCP(ブラウザ側の操作)と組み合わせることで、 「ブラウザで何が起きたか」と「サーバー側で何が起きたか」の両方をエージェントが横断的に調査できる

確認問題

  1. Next.js DevTools MCP が動作するために必須の Next.js バージョンはいくつですか?
  2. Next.js DevTools MCP と Playwright MCP を組み合わせる利点を説明してください。
