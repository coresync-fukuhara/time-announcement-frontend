Vitest の基礎

説明

Vitest はビルドツール Vite 上で動くテストランナー。describe/it/expect といった API は Jest とほぼ同じ(「Jest 互換」)なので、Jest の経験があればほぼそのまま書ける。 違いは中身のエンジンで、Vite の変換の仕組みをそのまま使うため TypeScript/ESM の設定がほぼ不要、 かつ起動・再実行が大幅に速い。

例えるなら、Jest が「毎回フルコースを作り直す」のに対し、Vitest は「材料を仕込んだまま次の一皿だけ作る」イメージ。 変更したファイルだけを再変換するので watch mode が特に速い。

コード例

// sum.ts
export function sum(a: number, b: number) {
  return a + b
}

// sum.test.ts
import { describe, it, expect } from 'vitest'
import { sum } from './sum'

describe('sum', () => {
  it('adds 1 + 2 to equal 3', () => {
    expect(sum(1, 2)).toBe(3)
  })
})

# 1回だけ実行(CI向け)
npx vitest run

# ファイル変更を監視して自動再実行(開発中)
npx vitest

ポイント

  • describe/it/expect/beforeEach は Jest とほぼ同じ書き方
  • デフォルトは watch mode。CI では vitest run を使う
  • TDD では「red(失敗するテストを書く)→ green(通す実装)→ refactor」のサイクルを回すため、 watch mode で保存するたびに即座に結果が見えることが重要
  • 設定ファイルは vitest.config.mts(詳細は次の概念でNext.js向け設定を扱う)
