Vercel Skills(skills.sh)

GitHub: github.com/vercel-labs/skills · 25,780 stars / 2,159 forks(2026-07-11時点) · 2026年1月14日作成、Vercel公式が発表・運用

説明

Skills.sh は Vercel が2026年1月に発表した、Skill(再利用可能なエージェント向け ワークフロー・ベストプラクティス・ドメイン知識)専門の配布エコシステム。 「Skill は文脈(context)における npm の瞬間だ」というコンセプトで、 毎回長い指示をコピペする代わりに、Skill を一度インストールしておけばエージェントが 必要な時に読み込んでくれる。

npx skills コマンドで完結するシンプルな CLI が特徴で、 Claude Code・Cursor・GitHub Copilot・Aider・OpenCode など30以上のエージェントに対応。

主なコマンド

npx skills find <keyword>     # キーワードで公開済みSkillを対話的に検索
npx skills add <package-name> # 指定したSkillをプロジェクトに直接インストール

実績・コミュニティの勢い

  • 発表からわずか6時間で、人気Skillのインストール数が20,000件を突破
  • Stripe が発表当日に自社Skillを公開するなど、大手企業の参加も早い
  • skills.sh 自体が「ディレクトリ + リーダーボード」として機能し、 実際のインストール数を基準に人気Skillを追跡できる

Microsoft APM との違い

観点Vercel SkillsMicrosoft APM
管理対象SkillのみSkill+Prompt+Instructions+Plugin+MCP+Hook
力点発見・共有・コミュニティの規模宣言的依存管理・ロック・組織ガバナンス
ロックファイル特になし(都度installする軽量モデル)apm.lock.yamlでSHA固定
後ろ盾Vercel(上場前提の大手フロントエンド企業)Microsoft(元は個人OSSの移管)

ポイント

  • 「信頼できるか」という観点では、Vercelという大手企業が主導し、 star数・実インストール数の実績も大きいため、現時点で最も裏付けの強い選択肢の一つ
  • ただしSkill専用であり、MCPサーバーの管理やロックファイルによる再現性保証までは範囲外
  • 「Skillの流通量・コミュニティの厚みを重視するか」「MCP/Hookも含めた依存管理・ガバナンスを重視するか」 で APM と使い分ける、あるいは併用する余地がある
