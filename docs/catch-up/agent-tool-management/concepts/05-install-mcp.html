Concept 05 / 06

install-mcp(軽量CLIインストーラー)

説明

install-mcp は、レジストリもホスティングも持たない、 「クライアントの設定ファイルへの追記」だけを自動化する最小構成のCLI。 16種類以上のクライアント(claude-desktop, cursor, cline, windsurf, VS Code, Zed, Warp など)に対応し、 パッケージ名を指定するだけでそのクライアント向けの設定を書き込んでくれる。

例えるなら、家具屋(レジストリ)やレンタル倉庫(ホスティング)は使わず、 「自分で選んだ家具を、指定した部屋の配置図に書き込むだけ」を代行してくれる作業員のようなもの。 何を入れるかは自分で決める前提で、置き場所への転記だけを楽にする。

コード例

# 基本形
npx install-mcp パッケージ名 --client claude-desktop

# OAuth認証やカスタムヘッダーが必要な場合も対応
npx install-mcp @playwright/mcp --client claude-code

メリット

  • アカウント登録・ホスティング契約が一切不要。ローカルで完結する
  • 対応クライアントの種類が非常に多い(16種類以上)
  • 「何を追加するか」を自分で完全にコントロールできる(隠れた仲介レイヤーがない)
  • 追加コストが発生しない(OSS・無料)

デメリット

  • レジストリ/検索機能がないため、「どんなMCPサーバーがあるか」は別途自分で調べる必要がある
  • セキュリティ検証(署名確認・隔離実行等)の機能は持たない。導入するサーバーの信頼性は利用者が判断する
  • Skill やAgentの管理機能はなく、あくまでMCPサーバーの設定追記のみ
  • チーム内での構成共有は、結局 `.mcp.json` 等の設定ファイルをコミットする従来の方法に頼ることになる

確認問題

  1. install-mcp が Smithery や Docker MCP Toolkit と比べて「持っていない」機能は何ですか?
  2. install-mcp のような最小構成のツールが向いている状況はどんな場合だと考えられますか?
