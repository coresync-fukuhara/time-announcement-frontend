Microsoft APM

GitHub: github.com/microsoft/apm · MIT ライセンス · 3,169 stars / 277 forks(2026-07-11時点) · 活発に開発中(直近プッシュも当日)

説明

APM(Agent Package Manager)は、AI コーディングエージェントの設定を npm や pip がコードの依存関係を扱うのと同じ発想で管理する OSS プロジェクト。 Skill・Prompt・Instructions・Plugin・MCP サーバー・Hook といった 「エージェントが必要とするあらゆる部品」を apm.yml に宣言し、 apm install 一発で誰の環境でも同じ構成を再現する。 01で見た「宣言的依存管理・再現性寄り」の代表格にあたる。

興味深い点として、このリポジトリは元々個人開発者(Daniel Meppiel)が danielmeppiel/apm として立ち上げた OSS で、同一のリポジトリのまま Microsoft org に移管されている(GitHub のリポジトリIDが一致することで確認済み)。 企業が単体で開発した公式ツールというより、コミュニティ発のプロジェクトが Microsoft に採用された経緯を持つ。

3つのコアプロミス

プロミス内容
ポータビリティ apm.yml に全設定を宣言。apm install で同一セットアップを全環境に再現。 transitive dependency resolution(npm/pip同様の推移的依存解決)にも対応
セキュリティ 隠れた Unicode ブロック等の脅威検出、apm.lock.yaml による整合性ハッシュ記録、 SBOM 出力対応、MCP サーバーの信頼境界管理
ガバナンス apm-policy.yml で組織が許可ソース・スコープを統制。 エンタープライズから単一リポまで段階的に適用可能

apm.yml / apm.lock.yaml の役割

  • apm.yml: プロジェクトが必要とする Skill/Prompt/Instructions/Plugin/MCPサーバー等を宣言する、 いわば package.json 相当のファイル
  • apm.lock.yaml: 解決済みの依存木をフルコミットSHA(40文字)で固定する、 package-lock.json 相当のファイル。今日 clone しても半年後に clone しても 同じエージェント挙動になることを保証する

対応クライアント

GitHub Copilot・Claude Code・Cursor・OpenCode・Codex・Gemini・Windsurf・Kiro の8クライアントに対応。 1つの apm.yml で複数クライアントの設定を横断できる点が特徴。

インストール

# Unix系
curl -sSL https://aka.ms/apm-unix | sh

# Homebrew / Scoop / pip 経由のインストールにも対応

ポイント

  • 02〜05の4ツールと違い、「レジストリで探す」より「宣言・ロック・再現・統制」に力点がある
  • Skill だけでなく MCP サーバーや Hook も含めて丸ごと依存管理できる点は、 02の Claude Code Plugins & Marketplace に近い発想だが、Claude Code専用ではなく 複数クライアント横断が前提
  • 組織的なガバナンス(apm-policy.yml)を持つ点が、他の類似ツールとの大きな差別化要因
