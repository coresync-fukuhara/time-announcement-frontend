Concept 02 / 06

Claude Code Plugins & Marketplace

説明

Claude Code の「プラグイン」は、Skill・Agent(サブエージェント)・Hook・MCP サーバー・LSP サーバー・ monitor をひとつのディレクトリにまとめた自己完結パッケージ。 「マーケットプレイス」はこのプラグインを配布する仕組みで、 /plugin install <name>@<marketplace> の一言でチームメンバーに同じ構成を配れる。

例えるなら、アプリ本体だけでなく「使い方の説明書(Skill)」「専門スタッフ(Agent)」 「自動反応するセンサー(Hook)」までをひとつの箱に詰めて配る、総合キットのようなもの。

コード例(構成イメージ)

my-plugin/
├── plugin.json
├── skills/
│   └── code-reviewer/SKILL.md
├── agents/
│   └── reviewer.md
├── hooks/
│   └── pre-commit.json
└── mcp/
    └── servers.json          # このプラグインが持ち込むMCPサーバー定義

# マーケットプレイスからインストール
/plugin install code-reviewer@my-team-marketplace

# インストール済みプラグイン一覧
/plugin list

メリット

  • Skill・Agent・Hook・MCP をまとめて1単位でバージョン管理・配布できる(今回の調査対象で唯一Skillも含む)
  • Claude Code 純正のため、追加のツールやアカウント登録が不要
  • プロジェクト設定を直接いじらずに、プラグインの有効/無効を切り替えられる
  • 2026年時点でマーケットプレイスの規模が大きく育っており(21,700+ Skills, 12,500+ MCPサーバー)、選択肢が豊富

デメリット

  • Claude Code 専用の仕組みのため、Cursor や VS Code Copilot など他クライアントでは使えない
  • マーケットプレイスが多数乱立しており、Skillと同様に「公式か・信頼できるか」の見極めが必要 (誰でもマーケットプレイスを作って公開できる)
  • MCPサーバー単体を隔離実行するようなセキュリティ機構(コンテナ分離等)は持たない
