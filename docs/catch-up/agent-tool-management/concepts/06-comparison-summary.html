比較まとめ(判断材料)

このページは特定のツールを推奨するものではありません。 プロジェクトの状況(チーム規模・対応クライアント・セキュリティ要件・コスト許容度)に応じて、 以下の一覧を判断材料としてご活用ください。

総合比較表

観点 Claude Code Plugins Smithery Docker MCP Toolkit install-mcp
管理対象 Skill+Agent+Hook+MCP+LSP MCPのみ MCPのみ MCPのみ
対応クライアント Claude Code専用 複数(Cursor/Claude Code/ChatGPT等) 複数(Claude Desktop/VSCode/Cursor等) 16種類以上
レジストリ/検索 あり(マーケットプレイス) あり(7,000+サーバー) あり(300+検証済み) なし
ホスティング なし(ローカル実行) あり(リモート実行可) なし(ローカルコンテナ実行) なし
セキュリティ機構 特になし OAuth自動管理 コンテナ隔離・署名検証・リソース制限(強) 特になし
料金 無料(個別プラグインの料金は提供者次第) 登録・CLIは無料/ホスティングは従量課金あり 無料(Docker Desktop自体は個人利用無料) 無料(OSS)
導入の手軽さ 中(マーケットプレイス選定が必要) 中(アカウント登録が必要な場合あり) 中(Docker Desktopが前提) 高(その場でコマンド一発)

メリット・デメリットの再掲

ツール最大のメリット最大のデメリット
Claude Code Plugins Skill含め丸ごと1パッケージで配布・共有できる Claude Code専用。他クライアントには使えない
Smithery ホスティングとOAuth自動化で運用の手間を減らせる サードパーティ経由。コストとデータ経路の確認が必要
Docker MCP Toolkit コンテナ隔離・署名検証でセキュリティ面が最も堅い Docker Desktopが前提。起動がやや重い
install-mcp 依存や契約が一切なく、その場ですぐ使える 発見・セキュリティ検証は自分で行う必要がある

判断の切り口(例)

  • Skill配布も含めて統一管理したい → 選択肢は実質 Claude Code Plugins のみ
  • 複数クライアントを横断して使う/チームの標準を揃えたい → Smithery か Docker MCP Toolkit
  • 信頼性の低い/出所不明なMCPサーバーも試したい、隔離実行を重視したい → Docker MCP Toolkit
  • とにかく手早く1つだけ追加したい、余計な依存を増やしたくない → install-mcp (本プロジェクトの .mcp.json への手動追記も実質これに近い方式)
