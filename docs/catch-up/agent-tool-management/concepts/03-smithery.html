Concept 03 / 06

Smithery

説明

Smithery は MCP サーバー版の「Docker Hub」を目指すサードパーティサービス。 7,000以上の MCP サーバーを検索できるレジストリに加え、 サーバー自体を Smithery のインフラ上で動かしてリモートエンドポイントとして提供する ホスティング機能を持つ。

さらに「Toolbox」という meta-MCP(MCPサーバーそのものをMCPで提供する入れ子構造)があり、 エージェントは Toolbox に接続するだけで、裏側でレジストリ内の複数サーバーに動的につながる。 1つずつ手動で `.mcp.json` に追記しなくても済む。

例えるなら、個別の店(MCPサーバー)を1軒ずつ契約する代わりに、 ショッピングモールの共通入り口(Toolbox)を1つ契約すれば、 中の色々な店に自動的に案内してもらえるイメージ。

コード例(利用イメージ)

// Smithery経由でホスティングされたサーバーに接続する例
{
  "mcpServers": {
    "toolbox": {
      "url": "https://server.smithery.ai/toolbox/mcp",
      "headers": { "Authorization": "Bearer <token>" }
    }
  }
}

メリット

  • レジストリ検索の情報(verified, useCount, owner等)で信頼性を判断しやすい
  • ホスティングを使えばローカルにインストール・運用しなくても良い(サーバーの実行責任をSmithery側に委ねられる)
  • OAuthの認証フローを自前実装しなくてよい(トークン自動更新・暗号化保管)
  • Cursor・Claude Code・ChatGPT など複数クライアントに対応

デメリット

  • ホスティング利用は無料枠を超えると有料。利用状況次第でコストが発生する
  • サードパーティのインフラを経由するため、機密性の高いデータを扱う場合は 経路・保存先のセキュリティポリシーを確認する必要がある
  • Skill やAgentの管理機能はなく、あくまでMCPサーバーが対象

確認問題

  1. Smithery の「Toolbox」が解決している課題を説明してください。
  2. Smithery を使う際に注意すべきコスト・セキュリティ面の懸念は何ですか?
