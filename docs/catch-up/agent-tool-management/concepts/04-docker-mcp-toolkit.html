Docker MCP Toolkit / Catalog

説明

Docker MCP Toolkit は Docker Desktop に組み込まれた機能で、 MCP サーバーをコンテナとして隔離実行することに力点を置く。 Docker MCP Catalog は 300 以上の検証済みサーバーをコンテナイメージとして提供するカタログで、 署名・SBOM(部品表)による真正性検証が行われている。

例えるなら、正体不明の作業員(MCPサーバー)を家(ホストマシン)に直接入れるのではなく、 ガラス張りの隔離作業室(コンテナ)の中だけで作業させ、 使える道具(CPU/メモリ)や持ち出せる荷物(ホストファイルシステムへのアクセス)を制限するイメージ。

本プロジェクトはすでに devcontainer(Docker)を使った開発環境を構築済みのため ([.devcontainer/devcontainer.json](../../../../.devcontainer/devcontainer.json))、Docker との親和性は高い。

コード例(セットアップの流れ)

# Docker Desktop の Catalog タブからサーバーを追加(GUI操作)
# その後、VSCodeなどのクライアントを接続する場合
docker mcp client connect vscode --profile my-profile

メリット

  • MCPサーバーは1CPU・メモリ2GB上限のコンテナに隔離され、ホストファイルシステムへは既定でアクセス不可
  • デジタル署名・SBOMによる真正性検証があり、「怪しいMCPサーバーを誤って実行してしまう」リスクを下げられる
  • 秘密情報(APIキー等)を含むツール要求を遮断する仕組みがあり、OAuth連携も自動管理
  • Claude Desktop・VS Code・Cursor など複数クライアントに MCP Gateway 経由で接続できる

デメリット

  • Docker Desktop(4.62以降)が前提。Docker自体を使わない/使えない環境では選べない
  • コンテナ起動のオーバーヘッドがあり、軽量なCLIツールに比べると起動が重い場合がある
  • Skill やAgentの管理機能はなく、あくまでMCPサーバーが対象

確認問題

  1. Docker MCP Toolkit がセキュリティ面で採っている「能動的」な対策を2つ挙げてください。
  2. 本プロジェクトの開発環境と Docker MCP Toolkit の親和性が高い理由を説明してください。
